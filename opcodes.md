# Pickle Opcode 参考指南

此表格列出了可用的所有 Pickle opcode，以及它们的十六进制表示、字符表示（如果适用）、引入该指令的协议版本以及对应的 Python 版本。

| 代码 (十六进制) | 字符 | 名称 | 协议 | Py 版本 | 已利用 | 用途 | 描述 |
| ------------ | ------ | ------ | ---------- | ---------- | ------ | ------ | ------------- |
| \x49 | `I` | INT | 0 | 所有版本 | ✅ | 构造整数/布尔常量 | 推入一个整数或布尔值。参数是一个以换行符结尾的十进制字面量字符串。最初可能意图是让它适合 Python 的短整数，但在 64 位机器上生成的 Pickle 可能会在 32 位机器上产生 Python 长整数。它与 LONG 的区别在于 INT 跳过了尾随的 'L'，并在可能时生成短整数。另一个区别是，当 2.3 引入布尔作为独立类型时，内置名称 True 和 False 也被添加到 2.2.2 中，映射为整数 1 和 0。为了双向兼容，True 被序列化为 `INT + "I01\n"`，False 为 `INT + "I00\n"`。正整数永远不会产生前导零。2.3 及更高版本的反序列化器会特例处理这些并返回 bool；早期的反序列化器忽略前导 "0" 并返回整数。 |
| \x4c | `L` | LONG | 0 | 所有版本 |  |  | 推入一个长整数。与 INT 相同，只是字面量以 'L' 结尾，且始终反序列化为 Python 长整数。尾随的 'L' 似乎没有实际用途。注意 LONG 在反序列化时耗费的时间与位数成平方关系（这是由于十进制转二进制的本质决定的）。协议 2 增加了线性时间（在 C 中；Python 中仍为平方时间）的 LONG1 和 LONG4 指令。 |
| \x53 | `S` | STRING | 0 | 所有版本 | ✅ | 构造字符串 | 推入一个 Python 字符串对象。参数是 repr 风格的字符串，带有引号和可能的转义字符。参数延伸到下一个换行符。这些通常使用提供给 Unpickler 构造函数的编码（或默认的 'ASCII'）解码为 str 实例。如果给定的编码是 'bytes'，则会解码为 bytes 对象。 |
| \x4e | `N` | NONE | 0 | 所有版本 | ✅ | 构造 None | 将 None 推入栈中。 |
| \x56 | `V` | UNICODE | 0 | 所有版本 | ✅ | 构造字符串 | 推入一个 Python Unicode 字符串对象。参数是 Unicode 字符串的 raw-unicode-escape 编码，因此可能包含嵌入的转义序列。参数延伸到下一个换行符。 |
| \x46 | `F` | FLOAT | 0 | 所有版本 | ✅ | 构造浮点数 | 以换行符结尾的十进制浮点数字面量。参数是 repr(a_float)，通常需要 17 位有效数字才能在往返转换中保持一致（对于 IEEE-754 双精度值是这样，Python float 在大多数机器上映射为此类值）。通常 FLOAT 不能跨机器传输无穷大、NaN 或负零，但在比 IEEE-754 双精度更高精度的机器上，它可能比 BINFLOAT 造成的损害更小。 |
| \x61 | `a` | APPEND | 0 | 所有版本 |  |  | 将一个对象追加到列表中。栈变化：`... pylist anyobject -> ... pylist+[anyobject]`。实际上 pylist 是原地扩展的。 |
| \x6c | `l` | LIST | 0 | 所有版本 | ✅ | 构造列表 | 根据栈顶的一段（从 markobject 之后开始）构建列表。markobject 之后的所有栈条目被放入一个 Python 列表中，该列表替换从 markobject 开始的所有栈内容。 |
| \x74 | `t` | TUPLE | 0 | 所有版本 | ✅ | 构造元组 | 根据栈顶的一段（从 markobject 之后开始）构建元组。例如：`... markobject 1 2 3 'abc' -> ... (1, 2, 3, 'abc')`。 |
| \x64 | `d` | DICT | 0 | 所有版本 | ✅ | 构造字典 | 根据栈顶的一段（从 markobject 之后开始）构建字典。例如：`... markobject 1 2 3 'abc' -> ... {1: 2, 3: 'abc'}`。栈中的内容交替作为键和值。 |
| \x73 | `s` | SETITEM | 0 | 所有版本 | ✅ | 字典单项赋值；属性赋值 | 向现有字典中添加一个键值对。栈变化：`... pydict key value -> ... pydict`。修改方式为 `pydict[key] = value`。 |
| \x30 | `0` | POP | 0 | 所有版本 |  |  | 丢弃栈顶项，栈长度减少 1。 |
| \x32 | `2` | DUP | 0 | 所有版本 |  |  | 将栈顶项再次推入栈中，实现复制。 |
| \x28 | `(` | MARK | 0 | 所有版本 | ✅ | 延长容器边界 | 将 markobject 推入栈中。markobject 是一个独特对象，被其他指令用来识别栈中包含变长对象的部分。 |
| \x67 | `g` | GET | 0 | 所有版本 | ✅ | 引用变量 | 从 memo 中读取对象并推入栈。要推入的 memo 对象索引由其后的换行符结尾的十进制字符串给出。BINGET 和 LONG_BINGET 是空间优化版本。 |
| \x70 | `p` | PUT | 0 | 所有版本 | ✅ | 保存变量 | 将栈顶存储到 memo 中。栈顶不弹出。索引由随后的十进制字符串给出。 |
| \x63 | `c` | GLOBAL | 0 | 所有版本 | ✅ | from module import name | 将一个全局对象 `(module.attr)` 推入栈。GLOBAL 后跟两个换行符结尾的字符串，分别是模块名和类名。实际上由 `find_class(module, class)` 返回的对象被推入栈。 |
| \x52 | `R` | REDUCE | 0 | 所有版本 | ✅ | 函数调用 | 推入由一个可调用对象（callable）和参数元组（argtuple）构建的对象。指令名称提醒了 `__reduce__()` 方法。栈变化：`... callable pytuple -> ... callable(*pytuple)`。 |
| \x62 | `b` | BUILD | 0 | 所有版本 | ✅ | 构造属性赋值 | 通过 `__setstate__` 或字典更新完成对象构建。如果对象有 `__setstate__` 方法，则调用之；否则参数必须是字典，且对象具有 `__dict__`。 |
| \x69 | `i` | INST | 0 | 所有版本 | ✅ | 构造类实例 | 构建类实例。这是协议 1 中 OBJ 指令的协议 0 版本。INST 后跟两个字符串给出模块名和类名。markobject 之后的所有栈元素被收集到元组中作为参数。 |
| \x2e | `.` | STOP | 0 | 所有版本 | ✅ | 字节码结束 | 停止反序列化机。每个 Pickle 都以该指令结束。弹出栈顶对象作为最终结果。 |
| \x50 | `P` | PERSID | 0 | 所有版本 |  |  | 推入一个由持久化 ID (persistent ID) 标识的对象。PERSID 参数是换行符结尾的字符串。反序列化器将该字符串传递给 `persistent_load()`，返回的对象推入栈。 |
| \x4a | `J` | BININT | 1 | 所有版本 | ✅ | 构造整数 | 推入一个 4 字节的有符号整数。这可以处理 32 位机器上全范围的 Python（短）整数，直接作为二进制字节（1 字节指令加上 4 字节整数）。如果整数是非负的且适合 1 或 2 字节，使用 BININT1 或 BININT2 可以节省空间。 |
| \x4b | `K` | BININT1 | 1 | 所有版本 | ✅ | 构造小整数 | 推入一个 1 字节的无符号整数。这是一种用于序列化 0-255 范围内非常小的非负整数的空间优化。 |
| \x4d | `M` | BININT2 | 1 | 所有版本 | ✅ | 构造小整数 | 推入一个 2 字节的无符号整数。这是一种用于序列化 256 到 `2**16-1` 范围内的小正整数的空间优化。0-255 范围内的整数也可以通过 BININT2 序列化，但使用 BININT1 可以多节省一个字节。 |
| \x54 | `T` | BINSTRING | 1 | 所有版本 |  |  | 推入一个 Python 字符串对象。有两个参数：第一个是 4 字节的小端无符号整数，表示字符串的字节数；第二个是相同数量的字节，直接作为字符串内容。通常使用 Unpickler 构造函数的编码（或默认 'ASCII'）解码为 str 实例。 |
| \x55 | `U` | SHORT_BINSTRING | 1 | 所有版本 |  |  | 推入一个 Python 字符串对象。有两个参数：第一个是 1 字节的无符号整数，表示字符串的字节数；第二个是相同数量的字节，直接作为字符串内容。通常解码为 str 实例。 |
| \x58 | `X` | BINUNICODE | 1 | 所有版本 | ✅ | 构造字符串 | 推入一个 Python Unicode 字符串对象。有两个参数：第一个是 4 字节小端无符号整数表示字节数。第二个是 UTF-8 编码的 Unicode 字符串数据。 |
| \x47 | `G` | BINFLOAT | 1 | 所有版本 |  |  | 以二进制形式存储的浮点数，包含 8 字节数据。通常比 FLOAT 编码节省一半以上的空间。通常 BINFLOAT 不能用于传输无穷大、NaN 或负零，如果指数超过 IEEE-754 双精度范围则抛出异常，并且精度保留不超过 53 位。 |
| \x5d | `]` | EMPTY_LIST | 1 | 所有版本 |  |  | 推入一个空列表。 |
| \x65 | `e` | APPENDS | 1 | 所有版本 |  |  | 将栈上的一段对象切片扩展到列表中。栈变化：`... pylist markobject stackslice -> ... pylist+stackslice`。实际上 pylist 是原地扩展的。 |
| \x29 | `)` | EMPTY_TUPLE | 1 | 所有版本 |  |  | 推入一个空元组。 |
| \x7d | `}` | EMPTY_DICT | 1 | 所有版本 | ✅ | 构造空字典（用于 BUILD） | 推入一个空字典。 |
| \x75 | `u` | SETITEMS | 1 | 所有版本 | ✅ | 下标赋值(模拟 `setitem`) | 向现有字典添加任意数量的键值对。栈中 markobject 之后的内容作为键值交替序列，添加到 markobject 下方的字典中。 |
| \x31 | `1` | POP_MARK | 1 | 所有版本 |  |  | 弹出栈中 markobject 及其以上的所有对象。当使用变长对象的指令完成时，使用 POP_MARK 清理栈。 |
| \x68 | `h` | BINGET | 1 | 所有版本 |  |  | 从 memo 中读取对象。索引由随后的 1 字节无符号整数给出。 |
| \x6a | `j` | LONG_BINGET | 1 | 所有版本 |  |  | 从 memo 中读取对象。索引由随后的 4 字节无符号小端整数给出。 |
| \x71 | `q` | BINPUT | 1 | 所有版本 |  |  | 将栈顶存储到 memo。索引为 1 字节无符号整数。 |
| \x72 | `r` | LONG_BINPUT | 1 | 所有版本 |  |  | 将栈顶存储到 memo。索引为 4 字节无符号小端整数。 |
| \x6f | `o` | OBJ | 1 | 所有版本 | ✅ | 构建类实例 | 构建类实例。这与 INST 非常相似。主要区别在于类对象从栈中获取，这使得如果创建同一类的多个实例时可以从 memo 中重复获取。 |
| \x51 | `Q` | BINPERSID | 1 | 所有版本 |  |  | 推入由持久化 ID 标识的对象。与 PERSID 类似，只是持久化 ID 从栈中弹出。 |
| \x8a |  | LONG1 | 2 | 2.3+ |  |  | 使用 1 字节长度的长整数。Python 长整数的一种更高效的编码方式。 |
| \x8b |  | LONG4 | 2 | 2.3+ |  |  | 使用 4 字节长度的长整数。Python 长整数的一种更高效的编码方式。 |
| \x88 |  | NEWTRUE | 2 | 2.3+ | ✅ | 构造 True | 将 True 推入栈中。 |
| \x89 |  | NEWFALSE | 2 | 2.3+ | ✅ | 构造 False | 将 False 推入栈中。 |
| \x85 |  | TUPLE1 | 2 | 2.3+ |  |  | 根据栈顶的一个元素构建一个单元素元组。从栈中弹出一个值并推回一个长度为 1 的包含该值的元组。即 `stack[-1] = tuple(stack[-1:])`。 |
| \x86 |  | TUPLE2 | 2 | 2.3+ |  |  | 根据栈顶的两个元素构建一个双元素元组。从栈中弹出两个值并推回一个包含它们的长度为 2 的元组。 |
| \x87 |  | TUPLE3 | 2 | 2.3+ |  |  | 根据栈顶的三个元素构建一个三元素元组。从栈中弹出三个值并推回一个包含它们的长度为 3 的元组。 |
| \x82 |  | EXT1 | 2 | 2.3+ |  |  | 扩展码。允许使用通过名称（通常是类）序列化的常用对象的注册表。旨在通过全局协商和注册流程，在整数和对象名称之间建立映射。EXT1 有 1 字节参数。 |

| \x80 |  | PROTO | 2 | 2.3+ |  |  | 协议版本指示符。协议 2 及以上版本的 Pickle 必须以此指令开始。参数是协议版本（2-255 的整数）。 |
| \x42 | `B` | BINBYTES | 3 | 3.0+ |  |  | 推入一个 Python bytes 对象。有两个参数：第一个是 4 字节小端无符号整数表示字节数，第二个是对应的字节数据。 |
